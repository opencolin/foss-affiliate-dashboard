Self-Hosting Setup Guide

Step-by-step complexity breakdown for each tool

Mautic Complex
StackPHP 8.0+, MySQL/MariaDB, Apache/Nginx, npm
Min RAM2 GB (512 MB minimum)
Shared HostingNot Supported
DockerAvailable
  1. Provision a VPS (2+ GB RAM recommended) with Ubuntu/Debian
  2. Install PHP 8.0+ with required extensions: xml, mysql, imap, zip, intl, curl, gd, mbstring, bcmath
  3. Install and configure MySQL/MariaDB database
  4. Install Apache/Nginx and configure virtual host
  5. Install Composer and Node.js/npm
  6. Clone Mautic repo, run composer install
  7. Configure .env.local with DB credentials and mailer settings
  8. Run php bin/console mautic:install
  9. Set up cron jobs for campaign processing and email sends
  10. Configure SSL certificate (Let's Encrypt recommended)
Affiliate note: Mautic does not have native affiliate management. You'll need third-party plugins (e.g., AffiliateWP bridge) or custom webhook/API integrations for affiliate tracking and commission management.
Raider Moderate
StackRust binary, MySQL
Min RAM256 MB (very lightweight)
Shared HostingNot Supported
DockerOfficial Image
  1. Pull official Docker image: docker pull valeriansaliou/raider:v1.2.3
  2. Or install via Debian APT repository (Debian 12 / Bookworm)
  3. Or install via Cargo: cargo install raider-server
  4. Create and configure config.cfg — set secret keys, DB URL, SMTP, branding
  5. Import raider.sql schema into MySQL database
  6. Generate secret tokens with openssl rand -base64 32
  7. (Optional) Configure Fixer API key for multi-currency support
  8. Start Raider: ./raider -c /path/to/config.cfg
  9. Integrate payment reporting API into your backend
Integration note: Raider requires your backend to POST payment events to its Track API when conversions occur. This is minimal code — a single HTTP POST with payment amount and currency.
Weferral Easy–Moderate
StackNode.js, React, PostgreSQL
Min RAM512 MB
Shared HostingPossible (Node hosts)
DockerCommunity Only
  1. Clone frontend: git clone https://github.com/WeferralHq/weferral.git
  2. cd weferral && npm install && npm start
  3. Clone backend API: git clone https://github.com/WeferralHq/weferral-api.git
  4. cd weferral-api && npm install && npm start
  5. Visit http://localhost:4100/setup to initialize DB and admin user
  6. Install Weferral Tracker JS snippet from separate tracker repo
  7. For production: npm run build:prod, deploy /dist to static host
Maturity note: Weferral is marked "not production ready" in its own README. The plugin framework and RBAC are listed as "coming soon." Evaluate carefully for production workloads.
